  <name>Task 1: Create POST /api/dev/seed route</name>
  <files>app/api/dev/seed/route.ts</files>
  <action>
Create app/api/dev/seed/route.ts as a Next.js Route Handler (App Router). This endpoint:

1. Uses the Supabase SERVICE ROLE key (process.env.SUPABASE_SERVICE_ROLE_KEY) with createClient from @supabase/supabase-js to bypass RLS. If the env var is missing, fall back to the anon key so the route still works in dev.

2. On POST: delete all existing rows in checkins where anonymous_message = 'DEMO_SEED' (so repeated calls are idempotent). Then insert ~50 synthetic check-ins with this shape:
   - city: cycle through ["Philadelphia", "New York", "Los Angeles", "Chicago", "Houston"]
   - mood: distribute across all 6 MOODS (import MOODS from @/lib/types to get the labels: "Blissful", "Happy", "Neutral", "Anxious", "Sad", "Overwhelmed")
   - weight: random float between 0.3 and 1.0 (Math.random() * 0.7 + 0.3), fixed to 2 decimal places
   - lat/lng: spread within each city's bounding box (Philadelphia center ~39.9526, -75.1652; offset by ±0.02 degrees randomly)
   - campus_name: pick from a small list per city (e.g. "Temple University", "Drexel University" for Philly; "NYU", "Columbia University" for NY; "UCLA", "USC" for LA)
   - user_id: use a fixed UUID string "00000000-0000-0000-0000-000000000001" (demo user, no auth needed)
   - anonymous_message: "DEMO_SEED" (used as the delete filter for idempotency)
   - created_at: spread over the last 24 hours (new Date(Date.now() - Math.random() * 86400000).toISOString())

3. Return JSON: { inserted: N, message: "Demo data seeded successfully" } with status 200.
4. On error: return { error: err.message } with status 500.

Mood distribution target: ~30% happy/blissful, ~25% neutral, ~25% anxious, ~20% sad/overwhelmed — makes the map visually interesting with warm and cool buildings.

City bounding boxes for lat/lng generation:
- Philadelphia: lat 39.93–39.97, lng -75.20 to -75.13
- New York: lat 40.70–40.75, lng -74.02 to -73.97
- Los Angeles: lat 34.02–34.07, lng -118.27 to -118.22
- Chicago: lat 41.86–41.90, lng -87.65 to -87.60
- Houston: lat 29.74–29.78, lng -95.40 to -95.35

No auth guard needed on this route — it's a dev/demo tool. Add a comment: "// DEMO ONLY - remove before production".
